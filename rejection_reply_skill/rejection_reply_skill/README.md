# 拒信助手 Skill (volcengine_rejection_reply)

## 1. 技能用途

本 Skill 是一个为火山引擎 HR 设计的专用工具，旨在帮助处理一个常见的招聘场景：当候选人收到拒信后，通过微信等即时通讯工具进一步询问未通过的具体原因时，能够快速、合规、得体地生成回复。

它能根据面试官的原始评价，自动生成三个不同语气（稳妥标准、温和鼓励、简洁直白）的回复版本，供 HR 选择使用。所有回复都严格遵守合规要求，避免法律风险和负面体验。

## 2. 输入与输出

### 输入参数

| 参数名 | 是否必填 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| `candidate_name` | 否 | `string` | 候选人的称呼，如“张同学”，用于生成更个性化的回复。 |
| `job_title` | 是 | `string` | 招聘的岗位名称，如“资深算法工程师”。 |
| `feedback_text` | 是 | `string` | 面试官的原始评价文本，是生成回复的核心依据。 |

### 输出结果

技能会返回一个 JSON 对象，包含三个键，分别对应三个版本的回复文案：

*   `version1`: **稳妥标准版**，礼貌清晰，最通用、最稳妥的回复。
*   `version2`: **温和鼓励版**，语气更柔和，侧重安抚与鼓励。
*   `version3`: **简洁直白版**，简短干脆，直接点明核心差距，但仍保持礼貌。

HR 可以根据与候选人的沟通氛围和个人风格，选择最合适的一版直接复制发送。

## 3. 如何导入和使用 OpenClaw

将此 Skill 导入到你的 OpenClaw 平台非常简单。

1.  **获取配置文件**：你可以使用本文件夹中的 `skill.yaml` 或 `skill.json` 文件。
2.  **在 OpenClaw 中创建 Skill**：
    *   进入 OpenClaw 的 Skill 管理界面。
    *   选择“创建 Skill”或“导入 Skill”。
    *   将 `skill.yaml` 或 `skill.json` 的内容**完整复制**并粘贴到配置区域。
3.  **配置大语言模型 (LLM)**：
    *   在 Skill 配置中，找到 `llm` 部分。
    *   确认 `model` 字段的值是你所在平台或组织授权使用的大模型名称（例如 `moonshot-v1-128k` 或其他内部模型）。如果不是，请修改为正确的模型名称。
4.  **保存并测试**：
    *   保存 Skill。
    *   在 OpenClaw 的对话界面，通过触发意图或直接调用该 Skill，并提供必要的输入参数（`job_title` 和 `feedback_text`）来进行测试。

## 4. 合规与数据安全注意事项

**请务必遵守以下核心原则，这是使用本 Skill 的前提：**

*   **最小必要数据原则**：在向 Skill 提供 `feedback_text` 时，请确保内容不包含超出必要范围的个人敏感信息。工具的目的是提炼岗位匹配度的相关信息，而非处理候选人隐私。
*   **短周期留存策略**：建议并要求平台侧配置极短的日志留存周期（如 7 天），且日志内容应脱敏，不记录候选人身份标识和面试评价全文。详细建议请参考 `COMPLIANCE_NOTES.md`。
*   **禁止自动外发**：本 Skill 的设计是**生成建议内容供人工审核**，**绝不能**配置为自动向任何外部渠道（包括但不限于邮件、IM）发送回复。所有对外沟通都必须由 HR 确认后手动发出。
